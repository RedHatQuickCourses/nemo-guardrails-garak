= Deploy EvalHub with the TrustyAI Operator

== Prerequisites

* You have cluster administrator privileges for your OpenShift cluster.
* You have installed the OpenShift CLI (`oc`) version 4.12 or later.
* You have the TrustyAI component in your OpenShift AI `DataScienceCluster` set to `Managed`.
* You have configured KServe to use `RawDeployment` mode.

== MLflow Deployment

. Create MLflow deployment `mlflow.yaml` configuration file in `lemonades` namespace.
+
[source,yaml,role=execute,subs="attributes,quotes"]
----
---
apiVersion: v1
kind: Namespace
metadata:
  name: *lemonades*
---
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: mlflow-pvc
  namespace: *lemonades*
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 5Gi
---
apiVersion: v1
kind: Service
metadata:
  name: mlflow
  namespace: *lemonades*
spec:
  selector:
    app: mlflow
  ports:
    - protocol: TCP
      port: 5001
      targetPort: 5001
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: mlflow
  namespace: *lemonades*
spec:
  replicas: 1
  selector:
    matchLabels:
      app: mlflow
  template:
    metadata:
      labels:
        app: mlflow
    spec:
      containers:
      - name: mlflow
        image: python:3.10-slim
        command:
        - /bin/bash
        - -c
        - |
          # 1. Create and activate a venv in /tmp (which is always writable)
          python -m venv /tmp/venv
          source /tmp/venv/bin/activate

          # 2. Install MLflow safely into the venv
          pip install mlflow

          # 3. Start the MLflow server
          mlflow server \
            --backend-store-uri sqlite:////mlflow/mlflow.db \
            --default-artifact-root /mlflow/artifacts \
            --host 0.0.0.0 \
            --port 5001
        ports:
        - containerPort: 5001
        volumeMounts:
        - name: mlflow-storage
          mountPath: /mlflow
      volumes:
      - name: mlflow-storage
        persistentVolumeClaim:
          claimName: mlflow-pvc
----

. Deploy the resources to your cluster by running the following command.
+
[source,bash,role=execute,subs="attributes,quotes"]
----
oc apply -f mlflow.yaml
----
+
```
namespace/lemonades configured
persistentvolumeclaim/mlflow-pvc created
service/mlflow created
deployment.apps/mlflow created
```

. Verify the MLflow is deployed.
+
[source,bash,role=execute,subs="attributes,quotes"]
----
oc logs -l app=mlflow -n lemonades
----
+
```

== Postgresql Deployment

. Create a postgres database `postgres.yaml` for EvalHub.
+
[source,yaml,role=execute,subs="attributes,quotes"]
----
---
apiVersion: v1
kind: Namespace
metadata:
  name: *lemonades*
---
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: postgres-pvc
  namespace: *lemonades*
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 5Gi
---
apiVersion: v1
kind: Service
metadata:
  name: postgresql
  namespace: *lemonades*
spec:
  selector:
    app: postgresql
  ports:
    - protocol: TCP
      port: 5432
      targetPort: 5432
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: postgresql
  namespace: *lemonades*
spec:
  replicas: 1
  selector:
    matchLabels:
      app: postgresql
  template:
    metadata:
      labels:
        app: postgresql
    spec:
      containers:
      - name: postgres
        image: postgres:15-alpine
        ports:
        - containerPort: 5432
        env:
        - name: POSTGRES_USER
          value: "evalhub"
        - name: POSTGRES_PASSWORD
          value: "evalhub123"
        - name: POSTGRES_DB
          value: "evalhub"
        - name: PGDATA
          value: "/var/lib/postgresql/data/pgdata"
        volumeMounts:
        - mountPath: /var/lib/postgresql/data
          name: postgres-data
      volumes:
      - name: postgres-data
        persistentVolumeClaim:
          claimName: postgres-pvc
----

. Apply the configuration.
+
[source,bash,role=execute,subs="attributes,quotes"]
----
oc apply -f postgres.yaml
----
+
```
namespace/lemonades unchanged
persistentvolumeclaim/postgres-pvc created
service/postgresql created
deployment.apps/postgresql created
```

. Verify the database is running and connection is successful.
+
[source,bash,role=execute,subs="attributes,quotes"]
----
oc exec -it deployment/postgresql -n lemonades -- psql -U evalhub -d evalhub -c "\conninfo"
----
+
```
You are connected to database "evalhub" as user "evalhub" via socket in "/var/run/postgresql" at port "5432".
```

== EvalHub Deployment

. Ensure you have configured KServe to use RawDeployment mode.
+
[source,bash,role=execute,subs="attributes,quotes"]
----
oc get configmap inferenceservice-config -n redhat-ods-applications -o jsonpath='{.data.deploy}'
----
+
```
{
  "defaultDeploymentMode": "RawDeployment"
}
```

. Create a Secret `evalhub-db-credentials.yaml` containing the `PostgreSQL` connection string.
The Secret must contain a `db-url` key with a valid `PostgreSQL` connection URI.
+
[source,yaml,role=execute,subs="attributes,quotes"]
----
apiVersion: v1
kind: Secret
metadata:
  name: evalhub-db-credentials
  namespace: lemonades
type: Opaque
stringData:
  db-url: "postgres://evalhub:evalhub123@postgresql.lemonades.svc.cluster.local:5432/evalhub"
----

. Apply the created `evalhub-db-credentials.yaml`.
+
[source,bash,role=execute,subs="attributes,quotes"]
----
oc apply -f evalhub-db-credentials.yaml -n lemonades
----
+
```
secret/evalhub-db-credentials created
```

. Create a generic secret with a model api key for your MaaS model.
+
[source,bash,role=execute,subs="attributes,quotes"]
----
oc create secret generic model-api-key --from-literal=api-key="<YOUR AIP KEY>" -n lemonades
----
+
```
secret/model-api-key created
```

. Create an `EvalHub` custom resource to deploy the service, such as `evalhub_cr.yaml`.
+
[source,yaml,role=execute,subs="attributes,quotes"]
----
apiVersion: trustyai.opendatahub.io/v1alpha1
kind: EvalHub
metadata:
  name: evalhub
  namespace: lemonades
spec:
  replicas: 1
  database:
    type: postgresql
    secret: evalhub-db-credentials
  providers:
    - garak
  collections:
    - safety-and-fairness-v1
  env:
    - name: MLFLOW_TRACKING_URI
      value: "http://mlflow.lemonades.svc.cluster.local:5001"
----

. Apply the custom resource to the cluster.
+
[source,bash,role=execute,subs="attributes,quotes"]
----
oc apply -f evalhub_cr.yaml -n lemonades
----
+
```
evalhub.trustyai.opendatahub.io/evalhub created
```

. Confirm that the evalhub pod is running.
+
[source,bash,role=execute,subs="attributes,quotes"]
----
oc get pods -l app=eval-hub -n lemonades
----
+
```

. Install the EvalHub SDK with CLI support.
+
[source,bash,role=execute,subs="attributes,quotes"]
----
sudo dnf install python3-pip -y
pip install "eval-hub-sdk[cli]"
----

. Configure the CLI to connect to your EvalHub server.
+
[source,bash,role=execute,subs="attributes,quotes"]
----
evalhub config set base_url https://$(oc get routes evalhub -o jsonpath='{.spec.host}' -n lemonades)
evalhub config set tenant lemonades
----
+
```
Set 'base_url' in profile 'default'
Set 'tenant' in profile 'default'
```

. Grant evaluation permissions by adding rolebindings.
+
[source,yaml,role=execute]
----
oc apply -f - <<EOF
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  name: evalhub-evaluator
  namespace: lemonades
rules:
  - apiGroups: [trustyai.opendatahub.io]
    resources: [evaluations, collections, providers]
    verbs: [get, list, create, update, delete]
  - apiGroups: [mlflow.kubeflow.org]
    resources: [experiments]
    verbs: [create, get]
---
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: evalhub-evaluator-binding
  namespace: lemonades
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: Role
  name: evalhub-evaluator
subjects:
  - kind: ServiceAccount
    name: evalhub-service
    namespace: lemonades
EOF
----
+
```
role.rbac.authorization.k8s.io/evalhub-evaluator created
rolebinding.rbac.authorization.k8s.io/evalhub-evaluator-binding created
```

+
[source,bash,role=execute,subs="attributes,quotes"]

evalhub config set token $TOKEN
